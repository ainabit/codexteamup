# ADR Index

Architecture Decision Records capture durable technical choices: what was chosen, why, what was rejected, and what this means for future work.

Rules:

- add a new ADR when a durable architecture rule changes;
- do not overwrite history into ambiguity;
- if a decision is superseded, mark the ADR as superseded and reference the replacement.
