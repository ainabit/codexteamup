# Company Bus Exploration

- Status: exploratory
- Owner: `ctu/architect`

## Goal

Evolve CTU from a single-project coordination bus into a controlled multi-project anchor model with explicit boundaries.

## First questions

- how does one project expose an anchor or endpoint to another?
- what stays read-only first?
- how are identities, allow-lists, and correlation scoped?
- how does the external exchange boundary participate?

## Current stance

Do not implement write-capable cross-project flows until restart reliability and acceptance confidence are solid.
